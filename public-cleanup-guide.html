A simple guide for keeping the GitHub Pages site clean and classroom-ready.

Public cleanup pass

The site works best when the root remains clean and classroom routes are obvious.

Current setting: GitHub Pages should remain set to main / root.

Keep these public files/folders

  • .nojekyll
  • index.html
  • assets/
  • cases/
  • data/
  • methods/
  • shared/
  • teaching/

Remove later after the site is stable

  • old ZIP packages
  • old deployment guide HTML/MD files
  • temporary standalone test files
  • duplicate top-level case HTML files if the /cases/ versions work
  • duplicate top-level JSON files if the /data/ versions work
