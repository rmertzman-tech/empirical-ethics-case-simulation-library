Simulation Readiness

Instructor preflight checklist

  1. Open each case page before class.
  2. Click every tab you plan to use live.
  3. Confirm that central tables render or that a static fallback is visible.
  4. Tell students when a panel is a teaching proxy rather than a verified empirical measure.
  5. Use the Key Terms Glossary when discussing Teaching FHRI, FHRI_max, IPS, T1/T2 repair, or source verification.
  6. Do not ask students to enter private information into public case pages.

Status Vocabulary

How to read the readiness labels

  • Classroom-ready: Can be used live with students with ordinary instructor framing.
  • Teaching proxy — use with instructor framing: Useful for class, but should not be treated as a certified measure or final empirical proof.
  • Needs static fallback: Any central table or result should still make sense if JavaScript fails or is not available.
  • Reviewer-facing verification pending: Use for publication, external review, or public claims only after source verification, formula replication, and limitations review.
Student-facing language

Suggested framing

This case page is a teaching simulation. It uses public evidence and model assumptions to help us ask better ethical questions. It is not a certified audit or final institutional judgment.

Do not enter private personal, family, medical, legal, employment, or financial information into public case tools.
