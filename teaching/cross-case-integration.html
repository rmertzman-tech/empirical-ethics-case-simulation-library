Design Rule

Simulation first. Evidence next. Private application last.

The Capability Lab remains the public classroom simulation space. The EECSL holds empirical case modules. The Navigator remains private and student-controlled. Phase H links the three without collapsing them into one overloaded app.

Pedagogical rhythm:

Start with a live classroom simulation, open the matching EECSL case, apply the active analysis toolbelt, then route personal application into the private Navigator or course notebook.

Active Analysis Toolbelt

This is the recommended button sequence after a public simulation run.

Open Simulation

Run the public classroom demonstration first.

Output: Initial pattern, failure, or repair signature.

Ethical Lens Audit

Ask what the system forms, respects, produces, distributes, exploits, and hears.

Output: Lens-specific ethical questions.

Hidden Costs

Identify who bears load, credibility burden, care burden, risk, delay, or loss.

Output: Hidden-cost inventory.

Repair Builder

Separate T1 structural repairs from T2 agent/process improvements.

Output: Repair proposal with claim type.

Open EECSL Case

Move from classroom simulation to empirical case study.

Output: Evidence-governed case module.

Navigator Bridge

Move from public case to private student application without exposing private data.

Output: Private reflection route.
